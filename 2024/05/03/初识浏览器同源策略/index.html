初识浏览器同源策略

2024/5/3 Web JS
什么是浏览器同源策略

同源策略是浏览器安全策略之一，用来限制一个源(origin)的文档或加载的脚本与另一个源进行交互。

同源

协议、IP、端口相同

不同源

协议、IP、端口有一个或多个不相同

跨域/异源问题产生的本质

由于浏览器同源策略的存在，会对不同源请求的响应进行检验，如果服务端没有配置同源策略规则或配置了但当前 Origin 不在响应范围之内，则抛出异常。
ps：如果没有浏览器参与，即使是不同源请求，也不会有跨域问题

跨域问题的解决方案
  • CORS 配置
  • 使用代理服务器
  • 使用一些标签如 script 进行内嵌
CORS 是什么

一种基于 HTTP 头的机制。该机制是允许服务器标识除了自己以外的其他 origin(域，协议和端口)可以访问加载这些资源，这样浏览器就可以访问加载这些资源了。
CORS 是需要浏览器和服务器同时支持的。服务器需要配置请求头信息，而浏览器发现 AJAX 请求(XMLHttpRequest)跨源，就会自动添加一些附加头的信息。
浏览器将 CORS 请求分成两类：简单请求和复杂请求

  • 简单请求：不会触发 CORS 预检请求。支持的方法有 GET、POST、HEAD，Content-Type 的类型限制为 text/plan、multipart/form-data、application/x-www-form-urlencoded
  • 非简单请求：正式请求之前会发起一次预检请求（preflight），请求方法为 OPTIONS，检验通过才会发起正式请求。增加了自定义请求头、使用 DELETE/PUT 方法、Content-Type 为 application/json 等场景下，会发起非简单请求
服务端 CORS 可配置的常见参数
  • Access-Control-Allow-Origin
  • Access-Control-Allow-Credentials
  • Access-Control-Request-Method
  • Access-Control-Request-Headers
  • Access-Control-Expose-Headers
  • Access-Control-Max-Age
